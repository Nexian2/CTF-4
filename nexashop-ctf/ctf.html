BakaCTF // NexaShop

Challenge Brief

Two-stage web exploitation challenge. Complete both stages to capture the flag.

Stage 1 — Credential Discovery XSS

The NexaShop staff portal is locked. Somewhere on this site, developer leftovers expose credentials — but they are encoded. Find them and decode them to gain access.

  • Explore the site and identify any input that may be unsafely rendered.
  • Craft a payload to expose hidden JavaScript state in the page.
  • The exposed value is encoded with three layers. Decode them in the correct order:
    Layer 1: ROT13  |  Layer 2: Base64  |  Layer 3: Caesar cipher (shift 3)
  • Decoding order: ROT13_decode → Base64_decode → Caesar_decode(shift=3)
  • The decoded output has the format username:password
Hint: Check the ?q= parameter on the search page. Look at how the query value is rendered into the DOM. The JS object you want is accessible as a property of window.

Stage 2 — Flag Extraction SQLi

Once inside the admin dashboard, navigate to Log Analysis. The filter input is vulnerable to SQL injection. A specific log entry contains the flag — normal filtering will not show it.

  • The backend constructs a raw SQL query using the filter input without sanitization.
  • Inject a payload that bypasses the WHERE clause to return all rows.
  • Look carefully through all returned log entries for the flag string.
  • Submit the flag in the box below the log table to complete the challenge.
Hint: Classic string-based SQLi. The query looks like: WHERE username = '[your input]'. Think about how to make the condition always true.
